## Editable Mode and Events

In editable mode, the component triggers a series of events that can be used to interact and respond to user actions. Additionally, using `bind:`, it is possible to link the component's content to a variable in the script, allowing automatic synchronization.

<div class="card preview rounded">
  <QuoteEditableExample />
</div>

<Code code={QuoteEditableExampleRaw} lang="svelte" />

### Available Events

- **`on:focus`**: Triggered when the component gains focus.
- **`on:keydown`**: Triggered when a key is pressed while the component is focused.
- **`on:input`**: Triggered whenever the user makes changes to the component's content.

### Bindings (`bind:`)

- **`bind:value`**: Allows synchronizing the editable content of the component with an external variable (in this case, `content`). Any changes to the component's content will update this variable, and any changes to the variable will update the component's content.
